EXECUTION INSTRUCTIONS
================================================================================

1. BACKUP DATABASE (CRITICAL):
   pg_dump -U postgres -d mizizzi_db > backup_$(date +%Y%m%d).sql

2. RUN INDEX CREATION SCRIPT:
   psql -U postgres -d mizizzi_db -f scripts/comprehensive_model_indexes.sql

3. VERIFY INDEXES CREATED:
   psql -U postgres -d mizizzi_db
   \d+ [table_name]  -- List indexes on table

4. CHECK INDEX STATISTICS:
   SELECT indexname, pg_size_pretty(pg_relation_size(indexrelid)) as size
   FROM pg_stat_user_indexes
   ORDER BY pg_relation_size(indexrelid) DESC;

5. ANALYZE PERFORMANCE:
   EXPLAIN ANALYZE SELECT ... (run your test queries)

MAINTENANCE OPERATIONS
================================================================================

DAILY CHECK:
  SELECT * FROM pg_stat_user_indexes
  WHERE idx_scan = 0;  -- Find unused indexes

WEEKLY MAINTENANCE:
  REINDEX INDEX CONCURRENTLY idx_name;  -- Rebuild fragmented indexes
  VACUUM ANALYZE table_name;

MONTHLY AUDIT:
  psql -c "SELECT schemaname, tablename, indexname
           FROM pg_indexes
           WHERE schemaname = 'public'

MONITORING & TROUBLESHOOTING
================================================================================

CHECK IF INDEX IS BEING USED:
  SELECT * FROM pg_stat_user_indexes WHERE indexrelname = 'idx_name';
  -- idx_blks_read > 0 means index is being used

FORCE SEQUENTIAL SCAN (for testing):
  SET enable_seqscan = off;
  EXPLAIN ANALYZE SELECT ...; -- Shows if index would be used

ANALYZE QUERY PLAN:
  EXPLAIN ANALYZE SELECT ...; -- Detailed execution plan

DROP UNUSED INDEX:
  DROP INDEX CONCURRENTLY idx_name;

REBUILD FRAGMENTED INDEX:
  REINDEX INDEX CONCURRENTLY idx_name;
